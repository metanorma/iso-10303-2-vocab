= ISO 10303-2 STEP Vocabulary

This repository hosts the https://metanorma.github.io/iso-10303-2-vocab/[ISO 10303-2 Vocabulary] website, built with the https://github.com/geolexista/glossarist-vocabulary-browser[Glossarist Concept Browser].

image:https://github.com/metanorma/iso-10303-2-vocab/actions/workflows/deploy.yml/badge.svg["Deploy Status", link="https://github.com/metanorma/iso-10303-2-vocab/actions/workflows/deploy.yml"]


== About

This information service is provided by **ISO/TC 184/SC 4/TF 1**, the task force responsible for managing the ISO 10303-2 vocabulary under https://www.iso.org/committee/54858.html[ISO Technical Committee 184, Subcommittee 4] (Industrial data).

ISO 10303 specifies the computer-interpretable representation and exchange of product data (STEP — STandard for the Exchange of Product model data). **ISO 10303-2** provides the vocabulary used throughout the ISO 10303 series, including terms, definitions, and EXPRESS entity type definitions.


== Datasets

iso10303-2-terms:: 685 terms and definitions from Clause 3 of ISO 10303-2, covering the vocabulary of product data representation and exchange across 32 thematic sections.

iso10303-2-express:: 7,363 EXPRESS entity concepts from Clauses 4 and 5 of ISO 10303-2, extracted from the integrated resources and application modules of the ISO 10303 STEP Library.


== Local Development

=== Prerequisites

* Node.js 20+
* Ruby 3.1+ with suma gem (for concept regeneration only)

=== Quick Start

[source,sh]
----
npm install
npm run dev
----

This generates concept data, starts a Vite dev server, and opens the site at
http://localhost:5173.


== Regenerating Concepts

Concepts are extracted from the private `iso-10303` source repository using
the `suma` tool. Set `ISO_10303_SRC` to point to your local clone:

[source,sh]
----
# One-time setup
gem install suma

# Extract EXPRESS concepts (requires access to iso-10303 repo)
ISO_10303_SRC=/path/to/iso-10303 rake concepts:express

# Copy term definitions
ISO_10303_SRC=/path/to/iso-10303 rake concepts:terms

# Or do both at once
ISO_10303_SRC=/path/to/iso-10303 rake concepts:all
----

=== Using Docker

If you have Docker, you can use the suma-docker image:

[source,sh]
----
docker run --rm -v /path/to/iso-10303:/src ghcr.io/metanorma/suma-docker:latest \
  suma extract-terms /src/schemas-smrl-part-2.yml /output --urn urn:iso:std:iso:10303:-2:ed-1:en
----


== Building the Site

[source,sh]
----
npm run build
----

The built site is output to `dist/`.


== CI/CD

* **deploy.yml** — Builds and deploys the site to GitHub Pages on push to `main`.
  Uses the committed `.datasets/` data (no private repo access needed).

* **regenerate-concepts.yml** — Extracts fresh concepts from the private
  `iso-10303` source repo and creates a PR with updated `.datasets/`.
  Runs weekly on Mondays or on manual trigger.
